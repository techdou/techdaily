Anthropic 介绍 Claude Code 模型选择与 effort level 选择策略

Anthropic 发布官方博客，解释 Claude Code 中模型选择与 effort level 的工作机制：模型决定能力，effort level则控制投入。官方建议多数任务使用默认 effort，并在出错时区分模型是能力不足还是投入不够。

Anthropic 通过官方博客发布了一篇关于 Claude Code 模型选择与 effort level 的指南。模型选择决定了处理请求的固定权重集合，即模型的整体能力范围；而 effort level 控制 Claude 在单次任务中投入的工作量，这不仅包括思考时间，还涵盖读取的文件数、执行的验证步骤以及多步任务的推进程度。官方建议，对于大多数任务应使用模型的默认 effort level，并将其视为基于工作类型的总体偏好而非逐任务调整；当 Claude 输出不符合预期时，应先检查上下文和提示的清晰度，然后判断是模型能力不足以解决问题（换用更大模型）还是 effort 投入不足（提高 effort level）。此外，官方将 Fable、Opus 和 Sonnet 分别定位为专家型、专家和通才，并指出在简单任务上小模型可节省成本，而复杂任务上大模型可能总成本更低且能完成小模型无法企及的任务。

Claude Code 团队发文介绍 Agent 的四种循环工作模式

Claude Code 团队近日发布文章，介绍了 Agent 的四种循环工作模式，并详细说明各类模式的应用场景、相关指令及代码质量管理策略。

Claude Code 团队发文将 Agent 的循环（loops）定义为重复执行工作直至满足停止条件的过程，并依据触发与停止方式将其分为基于回合、基于目标、基于时间和主动循环四种模式。文章分别说明了每种模式的适用任务，以及对应使用的 /goal、/loop 和 /schedule 等指令及其在本地或云端运行的区别。此外，团队还提出了一系列维持代码质量和优化 Token 用量的具体建议，包括使用 skills 验证机制、独立审查工具以及适当的用量审查指令。

Liquid AI 发布 Antidoom 消除推理模型重复循环

Liquid AI 发布 Antidoom，用于消除推理模型重复生成的“doom loop”现象。官方数据显示该方法可将模型循环率降至 1% 左右，相关代码已在 GitHub 公开。

Liquid AI 发布名为 Antidoom 的技术方案，旨在解决推理模型在处理复杂任务时陷入重复生成直到耗尽上下文窗口的“doom loop”失败模式。该方案基于 Final Token Preference Optimization (FTPO)，通过定位并重新训练引发循环的单一过度训练 token 来恢复模型连贯性。根据 Liquid AI 官方数据，早期 LFM2.5-2.6B 检查点的循环率从 10.2% 降至 1.4%，Qwen3.5-4B 在贪心采样下的循环率从 22.9% 降至 1%，且评估得分均有所提升。官方已在 GitHub 公开其生成、检测和 FTPO 训练器代码。

Google 发布 Gemma 4 技术报告

Google Gemma 团队发布 Gemma 4 技术报告。该系列首次融入思考模式，其中 12B 版本采用无编码器架构，可直接处理原始音频与图像。

Google Gemma 团队正式发布了 Gemma 4 技术报告，详细阐述了这一新一代开源原生多模态模型系列。该系列采用 Dense 与 Mixture-of-Experts 混合架构，参数规模从 2.3B 到 31B，并首次融入思考模式，允许模型在回答前生成推理轨迹；其中 12B 版本还采用了无编码器统一架构，可直接处理原始音频与图像。报告显示，Gemma 4 在 STEM、多模态与长上下文基准测试中性能大幅跃升，在人类评估任务中已可媲美更大的前沿开源模型。目前报告已公开，模型权重也已开源，但社区反馈中已出现微调兼容性、重复崩溃等实际使用问题。

美国比特币挖矿公司 TeraWulf 宣布与 Anthropic 签署 20 年租约，在其肯塔基州园区建设约 401 MW 的 AI 基础设施，预计产生 190 亿美元收入。

美国比特币挖矿公司 TeraWulf 宣布与 Anthropic 签署为期 20 年的 AI 基础设施租约，地点位于肯塔基州 Hawesville 的 Justified Data 园区。该园区可容纳约 401 MW 关键 IT 负载，初始容量预计于 2027 年下半年投入使用，并在 2028 年初达到满载，预计在初始租约期内为 TeraWulf 带来约 190 亿美元的合同租赁收入。

Anthropic起诉Abnormal侵权，创始人逐一反驳并称logo设计在先

Anthropic近日起诉网络安全厂商Abnormal AI，指其商标侵权并误导客户，要求没收全部收入。Abnormal创始人公开回应，称公司成立更早、logo设计在先，否认指控。

近日，Anthropic对Abnormal AI提起诉讼，声称后者的“/\”标志与Anthropic的“A\”相似，构成商标侵权和不正当竞争，并指控其误导安全客户，要求追缴全部收入与利润。Abnormal创始人兼CEO Evan Reiser发表博文公开回应，表示公司2018年成立，比Anthropic早三年；当前logo由ALINE于2021年4月设计完成，五年未变，且Anthropic并未在网络安全领域注册相关商标。Reiser另透露个人年消费近100万美元、公司年支出超1000万美元用于Anthropic服务，但并未收到提前告知。

微软被曝在 Excel 和 Word 中改用自研 MAI 模型以降低成本

据媒体报道，为降低 AI 成本，微软已开始在 Excel 和 Word 中使用自研 MAI 模型处理部分用户 prompt，减少对 OpenAI 和 Anthropic 模型的依赖。

据 Bloomberg 的报道，微软已开始部署成本节约策略，在 Excel 和 Word 程序中，使用其自研的 MAI 模型来响应一定比例的用户 prompt。过去微软曾表示 Office 365 的大部分功能由 OpenAI 和 Anthropic 模型支持，目前微软仍使用这些第三方模型。

支付宝 AI 开放平台正式开放，一次接入可跨终端提供服务

支付宝宣布正式上线支付宝 AI 开放平台，并面向企业开发者及代开发服务商开放邀测，支持将现有服务升级为 Agent、MCP 等 AI 能力，并通过“阿宝”实现多端分发与商业转化。

支付宝 AI 开放平台已正式上线，面向企业开发者和代开发服务商开放邀测。商家可通过平台将现有小程序和 API 等服务能力升级为 MCP、Skill 或 Agent，无需从零开发；接入后不仅能通过“阿宝”触达支付宝 10 亿用户，还可授权多类智能终端及大模型平台调用，实现跨端分发与统一管理，并借助平台提供的托管、交易和结算支持实现商业转化，同时通过统一协议转换降低研发成本。支付宝强调，上述升级不影响支付宝 App 上现有小程序服务，并将继续提供身份核验、支付安全等基础能力。

据路透社援引三位知情人士消息，DeepSeek正在开发一款专用于AI推理的芯片，以降低对其他企业芯片的依赖，该研发工作约一年前启动，目前仍处早期阶段。知情人士称，公司近几个月已私下增加芯片设计工程师招聘，并正与芯片设计、代工和存储厂商进行合作探讨。

报道称智谱探索为GLM模型定制AI芯片

据报道，智谱正与多家国内芯片设计公司进行早期洽谈，探讨为GLM模型定制专用AI芯片，以缓解模型用量激增带来的算力紧张。

据The Information独家报道，中国AI实验室智谱已与多个国内芯片设计公司展开初步讨论，考虑为GLM模型系列定制AI处理器。这一动向的背景是GLM-5.2模型日token用量在发布首周飙升最高27倍，计算资源成为关键瓶颈。目前洽谈处于早期阶段，智谱尚未选定合作伙伴，整个芯片设计与软件栈适配预计需要超过两年时间，拟开发的芯片类型为针对模型推理优化的ASIC，而非通用GPU。

报道称有关部门考虑限制海外访问国产AI模型

据路透社援引知情人士称，有关部门已与部分企业开会，拟限制海外访问最先进AI模型，并可能将技术泄露定为国家安全犯罪，但目前方案尚未确定。

据路透社报道，有关部门近期与阿里巴巴、字节跳动和智谱等公司开会，讨论限制海外访问本国最先进AI模型的措施，包括可能将AI技术泄露列为国家安全犯罪并限制对国内AI初创公司的投资。讨论涉及闭源和开放模型，但限制范围可能仅针对未来发布的产品，具体方案和实施时间尚未确定。有观点指出，此轮讨论主要聚焦海外收购问题，而非直接对模型使用施加控制。

NewAPI曝严重漏洞，官方要求紧急更新并复核账户余额

NewAPI项目核心贡献者近日发布紧急安全通知，称发现一项严重漏洞，要求所有用户立即更新至指定版本，并提醒更新后复核账户余额是否有异常增长。该漏洞与支付系统无关，但影响全部用户。

NewAPI项目核心贡献者通过开发者群组发布安全通知，称该项目存在一项严重漏洞，要求所有用户立即升级。开源版本最低需更新至 v1.0.0-rc.18，并建议使用 v1.0.0-rc.19；Horizon 版本则需升级至 0.6.0-rc.6 或更高。贡献者特别说明，此次修复与支付无关，但漏洞影响覆盖每一位用户，因此应立即行动。通知还提醒，更新后需对账户余额和增长记录进行复核，若发现异常增长应及时处置。
